# AXELIA Analytics セキュリティポリシー

AXELIA（以下「当チーム」といいます。）は、「AXELIA Analytics」（以下「本サービス」といいます。）において取り扱う情報を適切に保護するため、次のとおりセキュリティに関する方針を定めます。本ポリシーは、AXELIA Analytics 利用規約の一部を構成します。

# 第1条（基本方針）

当チームは、本サービスで取り扱う登録データ及び個人情報の機密性、完全性及び可用性を確保するため、組織的、人的、物理的及び技術的な観点から合理的な安全管理措置を講じ、その継続的な改善に取り組みます。当チームは、情報セキュリティが完全なリスクの排除を保証するものではないことを踏まえ、リスクを合理的に低減することを目指します。

# 第2条（アクセス制御）

1. 当チームは、チームごとに登録データを分離して管理し、利用者が自己のチームに割り当てられたデータのみにアクセスできるよう制御します。利用者は、他チームの登録データを閲覧・編集することはできません。
2. 当チームは、登録データ及び個人情報へアクセスできる当チームの従業者等を、業務上必要な者に限定し、必要最小限の権限を付与する運用に努めます。

# 第3条（認証）

当チームは、本サービスの利用にあたり、ID及びパスワードによる認証を実施します。
当チームは、パスワードについて、安全性に配慮した方法で管理するよう努めます。
当チームは、認証の強化（多要素認証の導入等）について、継続的に検討します。

※ 多要素認証等を実装した場合は、本条を実装内容に合わせて更新してください。実装前は「検討」の表現に留めています。

# 第4条（通信の暗号化）

当チームは、利用者のブラウザ等と本サービスとの間の通信について、TLS（Transport Layer Security）等の暗号化技術を用いて保護するよう努めます。

# 第5条（データの保護）

1. 当チームは、登録データ及び個人情報を、不正アクセス、漏えい、滅失又は毀損から保護するため、合理的な技術的措置を講じます。
2. 当チームは、業務上必要な範囲でデータのバックアップを行い、可用性の確保に努めます。

# 第6条（ログ管理）

当チームは、本サービスの運用状況の把握、不正アクセスの検知及びインシデント発生時の原因調査のため、アクセスログ等の記録を取得し、適切に管理します。

# 第7条（脆弱性への対応）

当チームは、本サービスを構成するソフトウェア等の脆弱性に関する情報の把握に努め、必要と判断した場合には、修正プログラムの適用その他の合理的な対応を行います。

# 第8条（インシデント対応）

1. 当チームは、情報セキュリティに関する事故（不正アクセス、情報漏えい等。以下「インシデント」といいます。）の発生又はそのおそれを認識した場合、速やかに事実関係の調査を行い、被害の拡大防止及び復旧のための措置を講じます。
2. 当チームは、利用者に影響を及ぼすインシデントが発生した場合、法令に従い、必要な範囲で利用者への通知及び関係機関への報告を行います。

# 第9条（利用者へのお願い）

本サービスの安全な利用のため、利用者には次の事項へのご協力をお願いします。

1. ID及びパスワードを適切に管理し、第三者に知られないようにすること
2. 推測されにくいパスワードを設定し、他サービスと同一のパスワードを使い回さないこと
3. 共用端末等でのログイン状態の放置を避けること
4. アカウントの不正利用のおそれを認識した場合は、速やかに当チームへ連絡すること

# 第10条（継続的改善）

当チームは、情報セキュリティを取り巻く環境の変化、技術の進展及び本サービスの状況を踏まえ、本ポリシー及び安全管理措置を定期的に見直し、継続的な改善に努めます。
